# Clinical Data Classification & Asset Discovery

## Objective
The goal of this project was to identify, classify, and govern sensitive healthcare-related data using Microsoft Purview data classification and asset discovery capabilities.

This implementation demonstrates how organizations can improve visibility into sensitive data stored across enterprise environments while supporting regulatory and compliance requirements.


# Data Discovery & Classification

Microsoft Purview was used to scan and classify structured datasets containing sensitive healthcare and personally identifiable information (PII).

The scan identified multiple sensitive data fields including:

- Full Names
- Email Addresses
- Dates of Birth
- Contact Information
- Healthcare-Related Data Fields

The classification process improves organizational visibility into where sensitive data exists and helps support data governance initiatives.


# Security Value

This implementation demonstrates how organizations can:

- Discover sensitive data across enterprise environments
- Improve data governance visibility
- Support compliance monitoring
- Reduce the risk of unauthorized data exposure
- Identify regulated healthcare and financial data
- Improve security auditing and reporting capabilities


# Compliance Alignment

This project aligns with:

- HIPAA
- NIST SP 800-171
- CMMC Level 2
- Microsoft Security Best Practices


# Key Governance Benefits

## Data Visibility
Organizations gain centralized visibility into sensitive data locations and classifications.

## Risk Reduction
Sensitive healthcare and personal data can be identified and protected before unauthorized exposure occurs.

## Compliance Support
Classification results support auditing, regulatory reporting, and governance initiatives within regulated environments.
