# Financial Data Loss Prevention (DLP) Policies

## Objective
The goal of this project was to implement Microsoft Purview Data Loss Prevention (DLP) policies to identify and protect sensitive financial information across Microsoft 365 environments.

This implementation demonstrates how organizations can reduce the risk of sensitive financial data exposure while improving governance and compliance monitoring.

---

# DLP Policy Configuration

Microsoft Purview DLP policies were configured to detect sensitive financial information using built-in sensitive information types and confidence-based detection logic.

The policy was designed to identify regulated financial data and support automated protection workflows across enterprise environments.

---

# Sensitive Information Types Detected

The DLP policy was configured to identify:

- Credit Card Numbers
- SWIFT Codes (for sending money internationally)
- International Banking Account Numbers (IBAN)
- U.S. Bank Account Numbers

Detection thresholds and confidence levels were configured to improve detection accuracy while reducing false positives.

---

# Security Value

This implementation demonstrates how organizations can:

- Detect sensitive financial data automatically
- Reduce accidental data exposure
- Improve regulatory compliance
- Strengthen data governance controls
- Improve visibility into sensitive organizational data
- Support enterprise security monitoring initiatives

# Key Protection Benefits

## Automated Detection
Sensitive financial information can be automatically identified using built-in Microsoft Purview detection capabilities.

Organizations can improve compliance reporting, auditing, and governance processes through centralized DLP policy management.
