# Microsoft Purview Sensitivity Labels

## Objective
The goal of this project was to implement Microsoft Purview sensitivity labels to classify and protect organizational data based on sensitivity and compliance requirements.

This implementation demonstrates how organizations can establish data governance controls and improve visibility into sensitive information across Microsoft 365 environments.

---

# Sensitivity Label Configuration

Sensitivity labels were created and published to support data classification and protection workflows across enterprise environments.

Labels were designed to align with common organizational data classification standards and regulatory compliance requirements.

The implemented label structure included:

- Public
- Internal Information
- Information Approved for Public Release
- Personal
- Confidential – Financial Data
- Confidential – PHI Data
- Highly Confidential – PCI Data

---

# Security Value

This implementation demonstrates how organizations can:

- Classify sensitive organizational data
- Improve governance and compliance visibility
- Support secure collaboration practices
- Reduce the risk of unauthorized data exposure
- Standardize data handling procedures
- Improve enterprise security posture

---

# Label Publishing Policies

Publishing policies were configured to deploy sensitivity labels across Microsoft 365 services and make labels available to organizational users and workloads.

This demonstrates centralized governance and policy management within Microsoft Purview environments.
